# Ubiquitous language

Use these terms precisely when working in or discussing this project.

## Development

- `Task` - used to decompose a large chunk of work into smaller, consumable
  chunks. This is intentionally the same term as in Linear tasks, and the
  TaskList tool builtin to Claude Code. ADRs, implementation plans, `/plan`
  mode, etc should _all_ use this term to decompose and map out work. Rejected
  alternatives: stage, phase, milestone, step.
